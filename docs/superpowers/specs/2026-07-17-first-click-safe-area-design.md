# 首次点击安全区设计

## 目标

改进随机布雷体验：玩家首次翻开格子时，在棋盘容量允许的情况下，点击格及其周围最多八个相邻格均不放置地雷。首次点击将因此展开一个可推理的起始区域，而不只是保证单个点击格安全。

该改进不改变固定地雷模式、难度参数、胜负规则或 UI 操作方式，也不引入第三方依赖。

## 方案选择

采用自适应安全区：标准棋盘保护完整的 3×3 邻域；只有当保护完整邻域后剩余格子不足以容纳配置的地雷时，才回退为仅保护首次点击格。

该方案兼顾体验与兼容性。内置初级、中级和高级难度都有足够空间使用完整安全区，同时保留现有高密度自定义配置，例如 2×2 棋盘放置 3 颗地雷。

未选择以下方案：

- 拒绝无法容纳完整安全区的高密度棋盘：规则最严格，但会破坏当前允许的合法配置。
- 只保护一部分相邻格：能扩大部分安全范围，但边界依赖额外的选择顺序，玩家难以理解和预测。

## 核心逻辑

修改 `src/game.mjs` 中的随机布雷流程：

1. 第一次调用 `revealCell` 时，使用现有邻域计算取得点击格的所有相邻格。
2. 构造包含点击格与相邻格的完整保护集合。
3. 从完整保护集合之外的格子中计算候选布雷位置。
4. 如果候选位置数量不少于地雷数，则仅在这些候选位置中随机布雷。
5. 如果候选位置不足，则重新使用除点击格以外的全部格子作为候选位置，保持“首次点击格必定安全”的现有保证。
6. 布雷完成后继续复用现有相邻地雷计数、空白区域展开和胜负判断流程。

固定地雷选项 `minePositions` 在创建游戏时已经完成布雷，因此不经过随机布雷流程，行为保持不变。

## 数据与边界行为

保护集合仅在首次随机布雷期间临时计算，不加入游戏状态，也不影响后续翻格或重新开始。

完整安全区会自然适配棋盘边缘：角落首次点击最多保护 4 个格子，边缘首次点击最多保护 6 个格子，内部首次点击最多保护 9 个格子。

随机函数仍由 `options.random` 注入。候选位置缩小后，现有的随机抽取方式和确定性测试能力保持不变。

## UI 与文档

页面结构和交互代码无需修改。用户仍通过普通左键或触控点击开始游戏，改进体现在首次随机布雷结果中。

更新 `README.md` 的功能列表，明确首次点击会优先获得无雷的相邻区域；同时说明极端高密度自定义棋盘会保留至少首次点击格安全的兼容行为。

## 测试

在 `tests/game.test.mjs` 增加或调整以下测试：

- 内部格首次点击后，点击格及八个邻格均不含地雷。
- 角落格首次点击后，点击格及三个有效邻格均不含地雷。
- 2×2 棋盘配置 3 颗地雷时回退为仅保护点击格，游戏仍可正常创建并在首次翻格后获胜。
- 固定地雷模式继续严格使用指定位置，不套用首次点击安全区。

运行完整的 `npm test` 验证所有既有和新增行为。通过本地静态服务器启动页面，手动检查首次点击能展开安全起始区域，且难度切换、重新开始、标旗和胜负流程保持正常。

## 非目标

- 不保证首次点击一定展开多个格子；相邻格没有地雷，但外围地雷分布仍可能使点击格带数字。
- 不修改固定地雷测试模式。
- 不改变地雷总数或对随机分布做重新平衡。
- 不增加设置开关或新的界面元素。
